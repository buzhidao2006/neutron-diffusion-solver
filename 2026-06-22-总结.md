## 关键点 1：源驱动 vs 本征值 — 反应堆物理的分界线

### 之前做的（源驱动）
L·φ = 已知源 → `spsolve` 一次搞定。这是 PDE 求解器。

### 反应堆实际（本征值）
A·φ = (1/k)·F·φ → 不知道 k，不知道 φ，不知道裂变源分布。需要迭代同时求解三者。

### 标准幂迭代
```python
s_0 = F·φ_0                    # 初始裂变源
for n in range(...):
    φ = A^{-1}·s_n             # 求解扩散
    s_{n+1} = F·φ              # 更新裂变源
    k_{n+1} = Σs_{n+1} / Σs_n   # Rayleigh 商
    收敛判定 |k_{n+1} - k_n| < tol
```

每次迭代做一次 `spsolve`。收敛速率取决于源迭代算子 H = F·A^{-1} 的次优势比 ρ_s。

典型 PWR 双群 2D 问题 ρ_s ≈ 0.93，196 次迭代到 1e-10。

---

## 关键点 2：Chebyshev 3-term 外推 — 工业加速的核心

### 理论
标准 PI 收敛速率 = ρ_s。如果 ρ_s = 0.93，每次迭代误差衰减 7%。

Chebyshev 对裂变源序列施加多项式滤波，3-term 递推：
```
s_{n+1} = ω_n·H·s_n + (1-ω_n)·s_{n-1}
```

ω_n 由 Chebyshev 系数递推确定：
```
ω_1 = 2/(2-ρ_s²)
ω_n = 1/(1 - ρ_s²·ω_{n-1}/4)    n ≥ 2
```

渐近收敛至 ω_opt = 2/(1+√(1-ρ_s²))。

理论加速后收敛速率：
```
ρ_acc = [ρ_s/(1+√(1-ρ_s²))]²
```

对 ρ_s = 0.93，ρ_acc ≈ 0.35。理论加速比 ≈ 14×。

### 实践（参数扫描）
对 60×60 双群问题进行 ω 参数扫描：

| ω | 迭代次数 |
|----|----------|
| 1.0 (标准 PI) | 180 |
| 1.2 | 118 |
| 1.3 | 92 |
| **1.4** | **65** |
| **1.45** | **43** |
| 1.5 | 45 |
| 1.6 | 50 |
| 1.7 | 78 |

**最优常量 ω ≈ 1.50，对应 ρ_s ≈ 0.94。**

实际加速比：196 → ~42 次 (4.7×)。未达理论极限 14×，原因是：
- 双群谱结构有多重次优势模（非单一 ρ）
- 源向量离散噪声
- 归一化操作引入的非线性

## 关键点 3：调试过程中的关键发现

### 误区 1：归一化破坏 Chebyshev 多项式
最初认为中间归一化（每步 ||s||=1）是非线性操作，会破坏 Chebyshev 多项式性质。

**实测结果**：归一化版（41 次）vs 不归一化版（46 次）— 归一化反而略快。因为归一化控制了数值幅值、避免了浮点溢出。

### 误区 2：ρ 应该从 k_eff 估计
k_eff 收敛速率 = ρ_s²（不是 ρ_s）。用错了导致 ω 偏低（1.24 vs 需要的 1.50）。

**修复**：从源向量差 `||s_{i+1}-s_i|| / ||s_i-s_{i-1}||` 直接估计 ρ_s。

### 误区 3：初始 ρ 估计可用
前 3 次迭代的局部 ρ 只有 ~0.5，远低于稳态值 0.93。过早估计导致加速失效。

**修复**：预热 15 步，等源向量收敛进入稳态后再估计 ρ_s。

## 关键点 4：ω 递推 vs 常量 ω

| 方法 | 迭代次数 | 优点 | 缺点 |
|------|----------|------|------|
| 标准 PI | 196 | 无参数 | 太慢 |
| 常量 ω=1.50 | 42 | 简单，最优 | 需预先参数扫描 |
| Chebyshev 递推 | 66 | 自适应 | ρ 估计误差导致次优 |

常量 ω=1.50 更快但对问题敏感。Chebyshev 递推更通用，ρ 估计的准确性决定加速比。

实际使用自适应 Chebyshev（3× 加速），保守但通用。
