## 关键点 1：k 更新公式的 bug — "为什么一个公式错，另一个公式对"

### 发生了什么

原代码的 k 更新公式让迭代结果**指数衰减到 0**，而不是收敛到真实的 k_eff。

### 错误公式

```python
k_new = k_old * sum(φ_new) / sum(φ)
```

**数学证明它错**：收敛时 φ_new = k_true · φ，代入得 k_new = k_old · k_true。要收敛需要 k_new = k_old，所以 k_true 必须 =1。但实际 k_true ≠ 1 → 永远不收敛，k 每步乘以 ~0.98，指数衰减到零。

### 正确公式

```python
k_new = sum(F @ φ_new) / sum(F @ φ)   # 裂变中子代际比
```

**物理含义**：k = 下一代裂变产生的中子数 / 当前代裂变产生的中子数。收敛时 φ_new = k_true · φ → k_new = k_true ✓

### 你能学到什么

**代码不是跑了就行，要验证对不对。** 一个看着像模像样的迭代公式，数学上可以证明它必然发散。物理学（裂变中子代际比）才是正确公式的来源，不是调参调出来的。

## 关键点 2：小网格 + 解析特征值 debug 法 — "怎么证明自己的代码没问题"

### 核心思想

不要用"看起来合理"来判断代码正确，用**数学定理**。

### 方法

三对角矩阵 (2, -1, -1) 的特征值有**解析公式**（正弦函数），不需要写代码，手算都能算出来。把代码构造的矩阵拿来对角化，和解析值对比——完全一致则矩阵没问题，不一致则 bug 在矩阵构造。

然后直接求 A⁻¹F 的特征值（`np.linalg.eigvals`），得到"标准答案"。最后跑迭代，看是否收敛到这个答案——不一致则迭代公式有 bug。

### 为什么不用大网格而是用小网格

小网格 (N=6) 的好处：
- 矩阵小到可以**手算验证**
- 运行速度快，秒出结果
- 特征值可以用 `np.linalg.eigvals` 直接算（大矩阵只能用迭代）

### 你能学到什么

**隔离变量**是通用的 debug 思想。一次只验证一个东西：
1. 先用解析公式验证矩阵 → 排除矩阵 bug
2. 再用直接特征值获得标准答案 → 有了对照基准
3. 最后对比迭代结果 → 锁定迭代公式问题

这个方法适用于所有数值计算代码，不只是核工程。

## 关键点 3：双群扩散 — "为什么要分快中子和热中子"

### 物理原因

单群把所有中子当同一能量，但实际物理中：

| | 快中子 (MeV) | 热中子 (0.025 eV) |
|---|---|---|
| 扩散能力 | 强 (D=1.2) | 弱 (D=0.4) |
| 被吸收概率 | 低 | 高 |
| 引起裂变概率 | 很低 | **高（主力）** |
| 主要去向 | 散射慢化→热群 | 裂变→产生快中子 |

这是一个**闭环耦合**：快群 → 散射 → 热群 → 裂变 → 产生快中子 → ...

### 代码结构

双群在矩阵上体现为 **2N×2N 分块矩阵**：

```
[A₁₁    0 ] [φ₁]       [F₁₁  F₁₂] [φ₁]
[A₂₁  A₂₂] [φ₂] = 1/k [  0    0 ] [φ₂]
```

- A₂₁ ≠ 0：快群散射到热群的耦合
- F₁₂ ≠ 0：热群裂变产生快中子的贡献
- A₁₂ = 0：热群不直接影响快群（只有通过裂变间接影响）

### 你能学到什么

- 多群方法不是"更精细"，而是**物理上根本不同**——单群无法描述快热谱差异
- 矩阵分块结构是处理耦合系统的通用模式（不只核工程，所有多物理耦合都类似）
- 吴宏春《核反应堆物理》第 4 章的多群扩散到这里就完全对应上了

## 关键点 4：临界尺寸 — "反应堆为什么不能太小"

### 物理

中子从诞生到被吸收，平均要走 √M² ≈ 7 cm。如果堆芯太小：

```
中子诞生 → 还没走多远就跑到堆芯外 → 泄漏 → 链式反应熄灭
```

增大堆芯 → 中子走同样的距离，但边界比例变小 → 泄漏减少 → k 增大 → 达到临界。

### 关键数值

| 参数 | 值 | 含义 |
|---|---|---|
| k_inf | 1.045 | 无穷大堆芯的 k（零泄漏） |
| M² | 47.9 cm² | 徙动面积，描述中子迁移距离 |
| L_crit | 102 cm | 临界尺寸 |

### 你能学到什么

- **临界条件** k_inf / (1 + M²B²) = 1 是反应堆物理最核心的公式之一，考研必考
- 几何曲率 B² = (π/L)² 连接了尺寸和泄漏
- 这个公式让你**从截面数据预测临界尺寸**，不需要跑迭代

## 关键点 5：临界硼搜索 — "PWR 怎么控制反应性"

### 物理

硼 (B-10) 是极强的热中子吸收体。PWR 把硼酸溶解在冷却剂里：

```
加硼 → Σ_a2 增大 → 热中子吸收变多 → k 下降
减硼 → Σ_a2 减小 → 热中子吸收变少 → k 上升
```

电站运行中：新燃料时硼浓度高（~1200 ppm），随着燃料消耗逐渐稀释硼，维持 k=1。

### 算法：外迭代 + 内迭代

```
外迭代 (二分法)      内迭代 (幂迭代)
┌──────────┐        ┌───────────┐
│ 猜一个 C_B │  ──→  │ 算 k(C_B)  │
│ 调整 C_B   │  ←──  │ 返回 k     │
│ 直到 k=1   │        └───────────┘
└──────────┘
```

这就是**嵌套迭代**——所有工程优化问题的标准解法。

### 关键数值

| 参数 | 值 |
|---|---|
| 临界硼浓度 C_B* | 293 ppm |
| 硼微分价值 | ~11 pcm/ppm |
| ΔΣ_a2 | 0.0029 cm⁻¹ |

### 你能学到什么

- **二分法**是最稳健的求根算法，不需要导数，保证收敛
- **嵌套迭代**是工程计算的骨架——外循环调设计参数，内循环求解物理方程
- 硼价值（pcm/ppm）是 PWR 运行的核心参数，复试说出来懂行
- 这就是 NECP 堆芯计算软件 Bamboo-C 的基本逻辑
