## 关键点 1：2D 的"墙"在哪里 — 为什么 1D 的方法不够用了

### 1D 的情况

60 个点 → 矩阵 120×120（双群）→ `np.linalg.solve` 瞬间出结果。

### 2D 的情况

60×60 网格 = **3600 个点** → 双群矩阵 **7200×7200**。如果还像 1D 那样存成普通矩阵 → 7200² ≈ **5200 万个元素**。

但每行（每个点）只和 4 个邻居有关 → 每行只有 **5 个非零**。其余 99.93% 全是零。

### 稀疏矩阵的核心思想

**只存非零元素**。5200 万 → 18000，内存省 700 倍。`scipy.sparse.spsolve` 利用稀疏性直接求解，速度和内存都在可承受范围内。

### 你能学到什么

这是 1D 玩具和 2D"真实"计算的**第一道墙**。真正的反应堆物理代码（NECP-X、PARCS、PANTHER）面对的矩阵是 10⁶–10⁷ 阶的，不用稀疏矩阵根本活不了。今天亲手碰到了这道墙。

## 关键点 2：5 点差分 vs 3 点差分 — 维度增加的本质

### 1D (3 点模板)

第 i 点的通量只依赖左右邻居：

```
φ[i-1]  —  φ[i]  —  φ[i+1]
```

每个方程 3 个系数 → 三对角矩阵。

### 2D (5 点模板)

每个点 (i,j) 依赖上下左右四个邻居：

```
              φ[i,j+1]   (上)
φ[i-1,j]  —   φ[i,j]   —  φ[i+1,j]   (左, 自己, 右)
              φ[i,j-1]   (下)
```

每个方程 5 个系数。矩阵不再三对角——x 方向的邻居紧挨着（偏移 ±1），但 y 方向的邻居隔了整整 Nx 个位置（偏移 ±Nx）。矩阵从"三个斜线"变成了"五条斜线，其中两条很远"。

### 边界条件的巧妙处理

和 1D 一样，用 **cell-centered 网格**——所有的格点都在区域内部，没有格点落在边界上。边界处通量天然为零（Dirichlet 条件），不需要手写边界处理代码。这是 1D 到 2D 唯一"不需要改"的部分。

### 你能学到什么

维度增加不是"多算一些"——它**改变了结构的本质**。3 对角可以用 Thomas 算法 O(N) 直接求解，5 点格式的矩阵不再是类似三对角的结构，Thomas 在 2D 不再成立。这就是为什么 2D 需要更复杂的求解器（sparse direct、SOR、CG 等）。

## 关键点 3：Kronecker 积 — "不用手写 5 点模板"的构造法

### 直觉

给一个 2D 矩形网格写 Laplacian，你其实只需要知道两件事：
1. **水平方向**的连接关系（谁和谁是左右邻居）
2. **垂直方向**的连接关系（谁和谁是上下邻居）

Kronecker 积的公式：

```
L_2d = I_y ⊗ L_x + L_y ⊗ I_x
```

其中：
- `L_x`：x 方向的 1D Laplacian（3 点模板，Nx×Nx）
- `L_y`：y 方向的 1D Laplacian（3 点模板，Ny×Ny）
- `⊗`：Kronecker 积，把两个矩阵"拼"成一个更大的矩阵

### 两个 Kronecker 项的含义

**`I_y ⊗ L_x`**：把 x 方向的 Laplacian 在 y 方向"复制"
- 思路：对于 y 方向的每一行，水平相邻关系完全相同 → 把 L_x 贴到对角线上

**`L_y ⊗ I_x`**：把 y 方向的 Laplacian 在 x 方向"铺开"
- 思路：对于 x 方向的每一列，垂直相邻关系完全相同 → 需要更复杂的排布，但 Kronecker 积自动处理

两者相加 = 完整的 2D 5 点 Laplacian。

### 为什么优雅

你不需要手写 3600×3600 的矩阵，不需要算每个点对应矩阵的哪一行哪一列——**两行代码，Kronecker 积自动把所有拓扑关系算对了**。即使 Nx ≠ Ny、hx ≠ hy（非均匀网格），也不需要改任何逻辑。

### 你能学到什么

- Kronecker 积是**笛卡尔网格**上的通用构造法——不只扩散方程，所有在矩形上做 PDE 离散的都能用
- 这个公式的美妙在于：1D → 2D → 3D 的扩展完全一样，3D 就是 `I⊗I⊗Lx + I⊗Ly⊗I + Lz⊗I⊗I`

## 关键点 4：2D 的解析验证 — "debug_demo 的方法论在 2D 依然成立"

### 验证 1：Laplacian 特征值

1D 三对角矩阵的特征值有解析公式：`λ_k = 4 sin²(kπ/(2(N+1)))`（debug_demo 的核心工具）。

2D 的 Laplacian 特征值就是**两个方向的叠加**：

```
λ_{p,q} = 4 sin²(pπ/(2(Nx+1))) + 4 sin²(qπ/(2(Ny+1)))
```

用这个公式对比数值对角化结果 → **误差 10⁻¹⁷**，矩阵构造确认无误。

### 验证 2：buckling 公式的自然延伸

1D：B² = (π/L)²
2D 方形：**B² = (π/Lx)² + (π/Ly)²**

buckling 近似的 k_eff 与 FDM 数值解：**偏差 0.09%**，物理上完全合理。

### 验证 3：网格收敛性

从 30×30 到 60×60，k_eff 的变化逐步减小（Δk 从 4×10⁻⁴ → 1.5×10⁻⁴），确认解在收敛。

### 你能学到什么

**隔离变量 + 解析对照**的 debug 方法论是普适的，和维度无关。1D 做到的验证（验证矩阵 → 验证迭代 → 验证物理），2D 一样可以做。
